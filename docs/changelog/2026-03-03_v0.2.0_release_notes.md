# OpenFlux v0.2.0 Release Notes

**Created:** 2026-03-03  
**Author:** Development Team  
**Status:** Published  

## Overview

OpenFlux v0.2.0 is a milestone release introducing **macOS cross-platform support**, **built-in Windows desktop automation (windows-mcp)**, **multi-session concurrent execution**, **browser session isolation**, and significant improvements to memory search and Agent collaboration rules. 22 files changed with ~3,000 lines of additions.

---

## New Features

### 1. macOS cross-platform support

**Impact:** Global (Rust backend, Tauri config, build system, frontend)  
**Description:** Single-branch codebase now supports both Windows and macOS builds.
- `gateway.rs`: Platform-aware Node binary (`node.exe` / `node`), tsx binary (`tsx.cmd` / `tsx`), PATH separator (`;` / `:`) via `cfg!(target_os)`
- `tray.rs`: macOS single-click tray icon support (Windows retains double-click)
- New `tauri.macos.conf.json`: Platform-specific Tauri config override for macOS resources
- New `scripts/build-gateway.sh`: macOS/Linux gateway build script with Apple Silicon/Intel architecture detection
- Frontend: Platform detection via `navigator.platform`, macOS titlebar left padding (70px) for native traffic light buttons
- Zero impact on existing Windows build flow — all changes are conditional branches

**New files:** `src-tauri/tauri.macos.conf.json`, `scripts/build-gateway.sh`  
**Modified:** `src-tauri/src/commands/gateway.rs`, `src-tauri/src/tray.rs`, `src/main.ts`, `src/styles/main.css`

---

### 2. Built-in Windows desktop automation (windows-mcp)

**Impact:** Tool system  
**Description:** Auto-injects `windows-mcp` MCP server at startup using built-in Python `uvx` (with system PATH fallback). Enables desktop UI automation, clipboard, process management, and system operations without manual MCP configuration.

**Modified:** `gateway/src/gateway/standalone.ts`, `gateway/src/utils/python-env.ts`

---

### 3. Multi-session concurrent execution

**Impact:** Agent core engine  
**Description:** Agent execution now supports concurrent sessions with per-session promise chain queuing. Same-session requests are automatically queued (transparent to users), while different sessions execute in parallel.
- New `sessionExecutionChains` map for per-session queuing
- New `activeExecutions` tracking map
- `sessionId` propagated through AgentLoopConfig to tools

**Modified:** `gateway/src/gateway/standalone.ts`, `gateway/src/agent/loop.ts`, `gateway/src/agent/manager.ts`

---

### 4. Browser session isolation

**Impact:** Browser tool  
**Description:** Browser tool now supports per-session page isolation. Different sessions can operate different browser tabs independently without interference.
- New `sessionPages` map for per-session page tracking
- `ToolExecutionContext` with `sessionId` passed to tool execute functions
- Tool type system updated with `ToolExecutionContext` interface

**Modified:** `gateway/src/tools/browser/index.ts`, `gateway/src/tools/types.ts`, `gateway/src/tools/registry.ts`

---

## Improvements

### 5. Memory search fallback and hint system

**Impact:** Memory system  
**Description:** 
- Lowered minimum relevance score from 0.3 to 0.05 for broader recall
- New fallback: When all searches return empty but database has memories, returns most recent entries
- New hint injection: Prompts LLM to use `memory_tool(action="list")` when search doesn't match but memories exist

**Modified:** `gateway/src/agent/memory/manager.ts`, `gateway/src/tools/memory.ts`

---

### 6. Tool collaboration rules (browser + windows-mcp)

**Impact:** Agent system prompts  
**Description:** Added explicit system prompt rules to prevent LLM from mixing `browser` and `windows-mcp` tools in the same operation, which causes connection conflicts and coordinate mismatches. Rules only activate when both tool types are available.

**Modified:** `gateway/src/agent/loop.ts`

---

### 7. Brave Search language code normalization

**Impact:** Web search  
**Description:** Fixed LLM frequently passing invalid `search_lang` values (e.g., `zh` instead of `zh-hans`). Added `BRAVE_SEARCH_LANG_MAP` for automatic normalization and improved parameter description.

**Modified:** `gateway/src/tools/web-search/index.ts`

---

### 8. Documentation

**Impact:** User-facing  
**Description:** Added comprehensive user guide and help documentation.

**New files:** `docs/planning/2026-03-02_OpenFlux用户使用指南.md`, `docs/planning/2026-03-02_OpenFlux使用帮助文档.md`

## Upgrade Notes

- Direct overlay installation from v0.1.6
- macOS build requires: Xcode CLI Tools, Rust toolchain, macOS Node.js binary in `src-tauri/node`
- Existing Windows configurations and workflows are fully backward compatible
- `windows-mcp` auto-injection requires `uv` (auto-installed via built-in pip) or system `uvx` in PATH
