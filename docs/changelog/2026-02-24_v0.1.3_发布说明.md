## Bug 修复

### 1. 修复引导设置后 LLM 对话返回 404 错误

**影响范围：** 核心功能（对话不可用）  
**触发条件：** 在引导设置中切换 LLM 供应商（如从 Moonshot 切到 DeepSeek）后，未手动填写 `baseUrl`  
**根因：** `handleSetupComplete` 中 `baseUrl` 使用条件赋值，切换 provider 后旧 provider 的 `baseUrl` 残留，导致请求发往错误端点  
**修复：** 改为优先使用 `payload.baseUrl`，其次从 `providers` 配置中查找该 provider 的默认 `baseUrl`，最后回退为 `undefined`（由 `factory.ts` 使用内置默认值）

**修改文件：** `gateway/src/gateway/standalone.ts` — `handleSetupComplete`

```diff
- if (payload.baseUrl) config.llm.orchestration.baseUrl = payload.baseUrl;
+ config.llm.orchestration.baseUrl = payload.baseUrl || config.providers?.[payload.provider]?.baseUrl || undefined;
```

（`orchestration` 和 `execution` 两个层级均已修复）

### 2. 修复 Router 托管开关首次设置后不可用

**影响范围：** Router 托管功能  
**触发条件：** 首次在引导设置中配置 Router 后，服务端标签页的"使用托管配置" toggle 不显示或点击无反应，需重启才正常  
**根因：** 两个子问题  

| 子问题 | 说明 |
|--------|------|
| **字段映射错误** | 向导发送 `appSecret`，但直接赋值给 `config.router`，未映射为 `RouterConfig` 所需的 `apiKey` 字段 |
| **缺少连接调用** | 保存 Router 配置后未调用 `routerBridge.updateConfig()`，Router 不会主动连接，LLM 托管配置不下发 |

**修复：** 正确构造 `RouterConfig` 对象（字段映射 + 补充默认值），并在保存后立即调用 `routerBridge.updateConfig()` 触发连接

**修改文件：** `gateway/src/gateway/standalone.ts` — `handleSetupComplete`

```diff
  if (payload.router?.enabled) {
-     (config as any).router = payload.router;
+     const routerConfig = {
+         url: payload.router.url || '',
+         appId: payload.router.appId || '',
+         appType: 'openflux' as const,
+         apiKey: payload.router.appSecret || '',
+         appUserId: '',
+         enabled: true,
+     };
+     (config as any).router = routerConfig;
+     routerBridge.updateConfig(routerConfig);
  }
```
