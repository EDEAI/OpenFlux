## Bug 修复

### 1. 修复定时任务交付物（成果物）检测缺失

**影响范围：** 定时任务核心功能  
**触发条件：** 定时任务执行后，Agent 成功生成文件，但前端成果物面板无法检测和显示  
**根因：** `executeScheduledAgent` 不广播 `chat.progress` WebSocket 事件（实时检测路径断裂），完成后也不提取/保存 artifacts（持久化路径断裂）  
**修复：**
- 在 `onToolStart`/`onToolCall` 回调中广播 `chat.progress` 事件，使前端能实时检测交付物
- 新增 `extractAndSaveScheduledArtifacts` 函数，任务完成后从工具调用记录中提取文件类 artifacts 并保存到 session
- 支持检测 `filesystem.write/copy/info`、`process/opencode` 的生成文件，含 stdout 正则回退检测

**修改文件：** `gateway/src/gateway/standalone.ts` — `executeScheduledAgent` + 新增 `extractAndSaveScheduledArtifacts`

---

### 2. 修复 Router 绑定弹窗置顶遮挡其他视图

**影响范围：** UI 交互  
**触发条件：** Router 未绑定时，绑定码输入弹窗固定在最上层，切换到设置或调度器视图时仍然遮挡  
**修复：** 在切换到设置视图、调度器视图时隐藏 Router 绑定 UI，返回聊天界面时恢复显示

**修改文件：** `src/main.ts`

---

### 3. 修复飞书多媒体消息预览

**影响范围：** Router 多媒体功能  
**触发条件：** 从飞书发送的图片/文件在客户端聊天界面无法预览  
**修复：** 实现多媒体消息气泡渲染和点击预览功能，集成现有文件预览机制

**修改文件：** `src/main.ts`、`src/styles/main.css`

---

### 4. 修复 Embedding 模型配置持久化与变更检测

**影响范围：** 长期记忆功能  
**触发条件：** 切换 Embedding 模型后重启应用，配置丢失；同维度模型切换不触发向量重建  
**修复：**
- Embedding 模型配置正确保存到 `server-config.json` 并在重启时恢复
- `MemoryConfig` 新增 `embeddingModel` 字段，持久化到 `memory_meta` 表
- 模型变更检测不再仅依赖维度，模型名称变化也触发重建

**修改文件：** `gateway/src/gateway/standalone.ts`、`gateway/src/agent/memory/manager.ts`、`gateway/src/agent/memory/types.ts`

## 升级说明

- 直接覆盖安装即可，无需额外配置
- 已有的 `server-config.json` 和 `openflux.yaml` 配置不受影响
- Embedding 模型配置会在首次启动时自动迁移
