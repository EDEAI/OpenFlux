## 新功能

### 1. 进程工具增强：kill / list 命令

**影响范围：** 沙盒进程管理  
**说明：** process 工具新增 `kill` 和 `list` 两个动作，支持终止和查看 Agent 通过 `spawn` 启动的后台进程，实现完整的进程生命周期管理。  
**技术细节：**
- 新增 `SpawnedProcess` 接口记录已 spawn 的进程（PID、命令、启动时间、会话 ID）
- `kill` 动作：通过 PID 终止指定进程
- `list` 动作：列出当前所有活跃的后台进程

**修改文件：** `gateway/src/tools/process/index.ts`、`gateway/src/tools/common.ts`

---

### 2. 新增"学习技能"预设工作流

**影响范围：** 工作流系统  
**说明：** 添加了第 9 个预设工作流"学习新技能"，支持从线上技能库（OpenClaw/ClawHub）搜索、下载并安装新技能，将其转化为本地可执行的工作流。  
**触发词：** "学习技能"、"学技能"、"安装技能"、"learn skill" 等

**修改文件：** `gateway/src/workflow/presets.ts`

---

## Bug 修复

### 3. 修复 Agent 路由空消息导致 LLM API 400 错误

**影响范围：** 多 Agent 路由  
**触发条件：** 会话历史中包含空消息时，LLM API 返回 400 错误  
**修复：** 加载会话历史时过滤空消息（`content.trim().length > 0`）

**修改文件：** `gateway/src/agent/manager.ts`

---

### 4. 修复仅上传附件不显示用户消息

**影响范围：** 聊天功能  
**触发条件：** 用户仅上传文件不输入文字时，会话列表中不显示消息记录  
**修复：** 当用户无文字输入但有附件时，自动生成 `[上传文件: filename]` 作为消息内容

**修改文件：** `gateway/src/agent/manager.ts`

---

## 优化

### 5. Typing 指示器重构

**影响范围：** 聊天 UI  
**说明：** 将加载动画从火焰黑洞旋转效果替换为三点跳动效果，并支持显示 LLM 意图/思考文本（截取前 120 字符）。切换会话后 typing 指示器位置正确恢复。  
**新增函数：** `ensureTypingPosition()`、`updateTypingText()`

**修改文件：** `src/main.ts`、`src/styles/main.css`

---

### 6. 进度卡片与 Typing 指示器位置协调

**影响范围：** 聊天 UI  
**说明：** Typing 指示器始终保持在进度卡片之前，避免切换会话后位置错乱

**修改文件：** `src/main.ts`
