## 文档概述

本指南面向 OpenFlux 终端用户，介绍如何通过自然语言与 AI 助手高效交互。  
你不需要记住任何命令或代码——**只需用日常语言描述你的需求**，OpenFlux 会自动选择合适的工具完成任务。

## 一、文件管理

适用于创建、读写、整理本地文件。

### 常用提示词

| 场景 | 推荐说法 |
|------|----------|
| 创建文件 | "帮我创建一个名为《周报》的 Word 文档" |
| 写入内容 | "把这段文字保存到 report.txt 里" |
| 读取文件 | "读一下桌面上的 data.csv，告诉我有多少条数据" |
| 查找文件 | "找一下最近修改的 Excel 文件" |
| 整理文件 | "把下载文件夹里的图片按日期分类整理" |
| 删除文件 | "删除 output/temp 里所有临时文件" |

### 进阶用法

- "把这 3 个 CSV 合并成一个 Excel，按日期排序"
- "统计一下 logs 文件夹有多少个文件，总大小是多少"
- "把所有 .png 图片批量重命名为 photo_001、photo_002…"

## 四、浏览器自动化

适用于需要像人一样操作网页的任务（登录、填表、购物、截图等）。

### 常用提示词

| 场景 | 推荐说法 |
|------|----------|
| 打开网页 | "帮我打开淘宝" |
| 搜索商品 | "在京东搜索 机械键盘 并截个图" |
| 截图保存 | "打开百度首页，截个图给我看看" |
| 填写表单 | "帮我打开这个在线表单，填写以下信息：…" |
| 登录网站 | "帮我登录小红书，账号是 186xxxx" |
| 连接浏览器 | "连接我已经打开的 Chrome 浏览器" |

### 注意事项

- 浏览器操作时，会优先连接你已打开的 Chrome（保留你的登录状态）
- 需要验证码时，助手会截图让你确认
- 涉及支付等敏感操作时，助手会提前征求你的同意

## 八、多 Agent 协作

OpenFlux 可以有多个专业助手，你可以让他们各展所长。

### 常用提示词

| 场景 | 推荐说法 |
|------|----------|
| 查看同事 | "你们团队都有谁？各自擅长什么？" |
| 指派任务 | "让营销助手帮我写一份推广方案" |
| 多人协作 | "同时让编程助手写数据分析脚本，让营销助手写分析报告" |
| 追问结果 | "刚才营销助手的方案完成了吗？" |

### 使用技巧

- 直接说"让 XX 助手去做 XX"即可，不需要记住任何 ID 或命令
- 可以同时安排多个助手并行处理不同任务
- 任务完成后会自动通知你结果

## 提示词写作技巧

### ✅ 好的提示词

| 特征 | 示例 |
|------|------|
| **具体明确** | "帮我搜索 2026 年 3 月的 AI 行业报告，整理前 5 条结果" |
| **提供上下文** | "我在做季度汇报，帮我生成一份包含营收、用户数的 Excel 报表" |
| **说明输出格式** | "用表格形式对比这 3 个方案的优缺点" |
| **分步骤复杂任务** | "先搜索竞品信息，再整理成对比文档，最后给出建议" |

### ❌ 不好的提示词

| 问题 | 示例 | 改进 |
|------|------|------|
| 太模糊 | "帮我查点东西" | "帮我查一下 Python 3.12 的新特性" |
| 缺少上下文 | "分析这个" | "分析这个 sales.csv 里的月度销售趋势" |
| 一次太多 | "做所有事情" | 拆分成 2-3 个明确任务 |

### 🎯 关键原则

1. **像对同事说话一样自然** —— 不需要任何特殊格式或指令
2. **越具体越好** —— 告诉助手你想要什么样的结果
3. **可以追问或修改** —— 不满意可以直接说"换一种方式"、"再详细一点"
4. **大任务拆小步** —— 先让助手完成一步，确认后再继续下一步
5. **善用附件** —— 直接拖拽文件到对话框，说"分析这个文件"
