## 1. 初次使用：设置向导

安装后首次启动 OpenFlux，会进入 4 步设置向导。

### 第 1 步：AI 助手

- **助手名称**：给你的 AI 起个名字（如"小明"、"Jarvis"），之后问它"你是谁"时会使用这个名字
- **人设（可选）**：描述助手的性格特点（如"你是一个温和细心的私人助理"），不设置则使用默认人设

### 第 2 步：AI 大脑

- 选择一个**模型供应商**（如 Moonshot、DeepSeek、OpenAI 等）
- 填入该供应商的 **API Key**
- 选择具体**模型**（如 Kimi K2.5、DeepSeek Chat 等）

> 💡 **推荐入门方案**：使用 Moonshot（Kimi K2.5），注册即送免费额度，体验最顺畅。

### 第 3 步：企业端连接（可选）

如果你的团队使用 NexusAI 云端平台，可以在这里输入账号密码连接。不连接也完全不影响使用。

### 第 4 步：远程操控（可选）

启用后可通过飞书、微信等平台远程与 AI 对话。跳过后可随时在设置中配置。

> 以上所有设置安装后都可以随时在「设置」面板中修改。

## 2. 界面总览

OpenFlux 界面分为以下区域：

```
┌────────────────────────────────────────────┐
│  标题栏：状态指示 · 成果物面板 · 主题切换     │
├──────┬─────────────────────────────────────┤
│      │                                     │
│ 侧   │        对话区域                      │
│ 边   │   （聊天消息、工具调用进度）            │
│ 栏   │                                     │
│      ├─────────────────────────────────────┤
│ 会话  │   输入框：文字输入 · 语音 · 文件拖放   │
│ 列表  │                                     │
│      ├─────────────────────────────────────┤
│ 定时  │   Debug 面板（可选）                  │
│ 设置  │                                     │
└──────┴─────────────────────────────────────┘
```

### 侧边栏

- **新对话按钮**（➕）：发起新的对话会话
- **会话列表**：所有历史对话，按"今天 / 昨天 / 更早"分组
- **搜索框**：搜索历史会话
- **定时任务**（⏰）：查看和管理定时任务
- **设置**（⚙️）：打开设置面板

### 标题栏

- **状态灯**：显示"就绪"、"连接中"、"思考中"等状态
- **成果物面板**（📎）：查看 AI 生成的所有文件（文档、代码、图片等）
- **主题切换**（🌙）：切换亮色 / 暗色主题

## 3. 对话交互

### 基本对话

在底部输入框输入文字后按 Enter 或点击发送按钮即可。

**你可以这样对话：**

| 场景 | 示例 |
|------|------|
| 知识问答 | "什么是量子计算？" |
| 翻译 | "把这段话翻译成英文" |
| 写作 | "帮我写一封求职邮件" |
| 分析 | "分析一下这份数据的趋势" |

### 发送图片

将图片直接**拖入对话框**即可发送，AI 可以识别和分析图片内容（需使用支持多模态的模型，如 Claude、GPT-4o、Kimi K2.5）。

### 消息操作

每条消息下方有快捷操作：

- **复制**：复制消息内容
- **朗读**（🔊）：用语音朗读这条回复
- **重试**：重新生成这条回复
- **停止生成**：AI 正在回复时点击可中断

### 代码块

AI 回复中的代码块会自动高亮显示，点击代码块右上角的"复制代码"按钮快速复制。

## 4. 智能体（Agent）

OpenFlux 内置了 3 个智能助手，会根据你的提问**自动选择**最合适的来回答：

### 通用助手

适合：闲聊、知识问答、翻译、咨询建议

**示例：**
- "帮我解释一下相对论"
- "推荐几本科幻小说"
- "这段英文翻译成中文"

### 编码助手

适合：写代码、处理文件、运行脚本、生成文档

**示例：**
- "帮我写一个 Python 爬虫脚本"
- "把这个 CSV 转成 Excel"
- "在 D:\project 目录下创建一个新项目"
- "帮我生成一份 Word 日报"

### 自动化助手

适合：浏览器操作、桌面控制、定时任务

**示例：**
- "帮我在京东搜索机械键盘"
- "每天早上 9 点帮我检查邮件"
- "打开浏览器搜索今天的天气"

### 手动指定 Agent

如果你想指定由某个助手来处理，可以用 `@` 语法：

```
@coder 帮我写一个排序函数
@automation 帮我打开百度搜索新闻
```

### 自定义人设

在 **设置 → 智能体** 中可以：

- **修改助手名称**：比如改成"小明"
- **设置全局角色设定**：定义助手的性格和行为规则，对所有助手生效
- **为每个 Agent 设置独立模型**：比如通用助手用便宜的模型，编码助手用强力模型

## 5. 定时任务

你可以用自然语言让 AI 创建定时任务，它会自动运行。

### 创建定时任务

直接在对话中告诉 AI 即可：

| 你说 | AI 会创建 |
|------|----------|
| "每天早上 9 点帮我检查日志有没有报错" | 每天 09:00 自动检查日志 |
| "每周一早上发一份工作周报" | 每周一定时生成周报 |
| "5 分钟后提醒我开会" | 5 分钟后弹出提醒 |
| "半小时后帮我备份数据库" | 30 分钟后执行一次备份 |
| "每隔 2 小时检查一下服务器状态" | 每 2 小时定期检查 |
| "明天下午 3 点提醒我打电话" | 指定时间一次性提醒 |

### 触发类型说明

| 类型 | 说明 | 使用场景 |
|------|------|---------|
| **周期性（cron）** | 按固定时间表重复执行 | "每天 9 点"、"每周一" |
| **固定间隔（interval）** | 按固定时间间隔重复执行 | "每隔 2 小时"、"每 30 分钟" |
| **一次性（once）** | 只执行一次 | "5 分钟后"、"明天下午 3 点" |

### 管理定时任务

- 点击侧边栏的 **定时任务**（⏰ 图标）查看所有任务
- 每个任务显示：名称、状态、下次执行时间、已执行次数
- 可以对任务进行：**暂停** / **恢复** / **立即执行** / **删除**
- 点击任务可查看**执行记录**

### 定时任务实战举例

**例 1：每日晨报**

> 你："每天早上 8 点帮我搜索今天的科技新闻，整理成简报发给我"

AI 会创建一个每天 08:00 触发的定时任务，到时间后自动搜索新闻并生成简报。

**例 2：定时监控**

> 你："每 30 分钟检查一下 D:\logs\app.log 有没有 ERROR"

AI 会创建一个每 30 分钟触发的任务，自动读取日志并检查错误。

**例 3：延时提醒**

> 你："1 小时后提醒我给客户发邮件"

AI 会创建一个 60 分钟后触发的一次性任务。

## 6. 工作流

工作流是 AI 自动执行的**标准化流程**。当你的请求匹配到某个工作流时，AI 会按照预设步骤自动完成。

### 内置工作流

OpenFlux 预置了 9 个常用工作流：

#### 📁 项目初始化

**触发方式**：说"帮我创建一个新项目" / "初始化项目"

会自动创建完整的项目结构：README、源码目录、配置文件（package.json/tsconfig.json）、.gitignore 等。

**示例**：
> "在 D:\projects 下创建一个叫 my-app 的 Node 项目"

#### 🐛 系统化 Bug 修复

**触发方式**：说"帮我修复这个 bug" / "排查问题" / "debug"

按照"定位→分析→修复→验证"四步流程系统化排查。

**示例**：
> "D:\project\src\app.ts 里有个报错，帮我排查修复"

#### 🔍 代码审查

**触发方式**：说"帮我审查这段代码" / "review 一下" / "检查代码质量"

自动读取代码、检查结构、分析依赖、总结问题。

**示例**：
> "帮我审查一下 D:\project\src\utils.ts 的代码质量"

#### 🚀 部署前检查

**触发方式**：说"发布前检查" / "部署检查" / "上线前确认"

执行标准上线检查清单：依赖检查→构建测试→配置验证。

**示例**：
> "D:\my-project 准备上线了，帮我做一下发布前检查"

#### 📂 批量文件处理

**触发方式**：说"批量处理文件" / "批量重命名"

扫描目录、匹配文件、批量执行操作。

**示例**：
> "帮我把 D:\photos 里的所有 jpg 文件批量重命名"

#### 📝 日报生成

**触发方式**：说"帮我写日报" / "生成工作日报" / "今日总结"

扫描工作目录、提取关键信息、自动生成 Word 格式日报。

**示例**：
> "根据 D:\work 里今天修改的文件，帮我生成一份日报"

#### 📊 数据提取与合并

**触发方式**：说"合并这些 Excel" / "数据汇总" / "提取数据"

从多个 Excel/CSV 文件中提取数据，合并到一个新文件。

**示例**：
> "把 D:\data 下所有 Excel 文件合并到一个总表里"

#### 🗂️ 文件整理

**触发方式**：说"帮我整理这个目录" / "文件分类" / "归类文件"

按文件类型自动分类到对应子目录（文档/图片/视频/...）。

**示例**：
> "帮我把 D:\downloads 里的文件按类型整理一下"

#### 📚 学习技能

详见下一章 [学习技能（Skill）](#7-学习技能skill)。

## 7. 学习技能（Skill）

Skill 是 OpenFlux 的一大亮点：你可以让 AI **永久学会新技能**，学会后随时可以调用。

### 什么是 Skill？

Skill 就像是教 AI 一套专业方法论。学会后，AI 在遇到相关任务时会自动使用这套方法来处理，而不是"临场发挥"。

### 如何让 AI 学习技能

直接说即可：

| 你说 | AI 会做什么 |
|------|------------|
| "学习深度研究技能" | 从网上搜索并安装"深度研究"技能 |
| "学一下做 PPT 的方法" | 搜索 PPT 制作技能，学会后可以帮你做 PPT |
| "安装数据分析技能" | 安装专业的数据分析方法论 |
| "学习写论文的技能" | 学会学术论文写作的标准流程 |

### 学习过程

1. **在线搜索**：AI 先在 GitHub 技能库（OpenClaw/ClawHub）中搜索现成的技能
2. **下载安装**：找到匹配的技能后自动下载并安装到本地
3. **自创技能**：如果网上没有现成的，AI 会基于自身知识**自行创造**一套技能
4. **持久保存**：技能保存在本地，重启不丢失，随时可用

### 学会后如何使用

学会的技能会转化为工作流，之后你只需要用自然语言描述任务，AI 就会自动调用已学的技能：

> 学习前："帮我研究一下量子计算" → AI 可能随意回答
>
> 学习"深度研究"技能后："帮我深度研究量子计算" → AI 按专业的研究流程执行：确定研究范围→文献搜索→信息整理→观点分析→生成报告

### 查看已学技能

对 AI 说："列出所有工作流" 或 "我学过哪些技能"，AI 会显示已安装的所有技能。

### 技能使用举例

**例 1：学习"深度研究"技能**

> 你："学习深度研究技能"
>
> AI：✅ 已学会「深度研究」！你可以让我对任何主题进行专业的深度研究，我会按照"调研框架→资料搜集→多角度分析→报告撰写"的流程进行。
>
> 之后你说："深度研究一下新能源汽车市场趋势"

**例 2：学习"PPT 制作"技能**

> 你："学一下怎么做PPT"
>
> AI：✅ 已学会「PPT 制作」！我可以帮你制作专业的演示文稿。
>
> 之后你说："帮我做一份关于年度销售总结的 PPT"

**例 3：自创技能**

> 你："学习如何写小红书文案"
>
> AI：（网上没找到现成技能，自动创造）✅ 已学会「小红书文案写作」！我会按照"选题→标题撰写→正文编写→标签优化"的流程帮你创作。

### 在设置中管理技能

在 **设置 → 智能体 → 技能** 区域，你还可以：

- **手动添加技能**：点击"添加技能"，输入标题和内容（Markdown 格式）
- **编辑已有技能**：修改技能的指令内容
- **启用/禁用技能**：暂时关闭某个技能
- **删除技能**：不再需要时删除

技能内容支持 Markdown 格式，你可以详细描述：
- 操作步骤和流程
- 专业知识和规则
- 输出格式要求
- 注意事项

## 8. 长期记忆

OpenFlux 具备长期记忆能力，能记住你们之间的重要对话内容。

### 记忆如何工作

- AI 会自动把对话中的**有价值信息**保存为"记忆卡片"
- 下次你提问时，AI 会自动搜索相关记忆，结合历史信息回答
- 例如上周你告诉它你的工作偏好，本周它依然记得

### 记忆管理

在 **设置 → 记忆管理** 中可以：

- **查看所有记忆**：浏览 AI 保存的记忆卡片
- **搜索记忆**：输入关键词搜索（支持语义搜索，不需要精确匹配）
- **按类型筛选**：Micro 卡片（细碎信息）/ Mini 卡片（单条知识）/ Macro 卡片（综合总结）/ 主题
- **删除单条记忆**：点击卡片上的删除按钮
- **清空所有记忆**：一键清除（不可恢复！）
- **查看系统信息**：记忆总数、数据库大小、向量维度等

### 记忆蒸馏

系统会在深夜（默认 02:00-06:00）自动进行"记忆蒸馏"：

- 清理低质量的冗余记忆
- 合并高度相似的重复记忆
- 保留有价值的核心信息

你也可以点击 **⚡ 手动蒸馏** 按钮立即执行。

## 9. 语音交互

### 语音输入

点击输入框旁的 **麦克风按钮**（🎤），对着麦克风说话，AI 会自动识别成文字并发送。

### 语音对话模式

点击 **实时语音对话** 按钮进入沉浸式语音模式：

- 点击中央按钮开始说话
- AI 识别后自动回复并用语音朗读
- 说话可打断 AI 的朗读
- 点击退出按钮回到文字模式

### 自动朗读

在 **设置 → 客户端 → 语音** 中开启"自动朗读回复"后，AI 每次回复都会自动用语音朗读。

### 语音角色

可以在设置中选择不同的语音角色，更换朗读的声音。

## 10. 浏览器自动化

OpenFlux 可以帮你操控真实的浏览器，完成网页上的各种任务。

### 能做什么

| 场景 | 示例 |
|------|------|
| 搜索信息 | "帮我在百度搜索今天的热点新闻" |
| 网购比价 | "在京东和淘宝搜索 Switch 游戏机，比较哪个便宜" |
| 表单填写 | "帮我打开这个网址，填写注册表单" |
| 数据抓取 | "帮我抓取这个页面上所有商品的价格" |
| 自动化操作 | "登录 GitHub 查看最新的 Issue" |

### 使用方式

直接对 AI 说你想在网页上做什么，AI 会自动打开浏览器执行。默认会显示浏览器窗口，让你看到操作过程。

### 搜索功能

除了浏览器操作，AI 还能直接搜索互联网获取最新信息：

> "帮我搜索量子计算的最新进展"

AI 会使用 Brave Search 搜索引擎查找信息并整理给你。

## 11. 文件预览与成果物

### 文件预览

AI 生成或提到的文件可以直接在应用内预览：

- **文档**：Word (.docx)、Excel (.xlsx)、PDF
- **代码**：各种编程语言（语法高亮）
- **图片**：点击查看大图
- **其他**：点击"用默认应用打开"按钮

每个预览窗口提供：
- **用默认应用打开**：调用系统默认程序打开
- **在文件夹中显示**：在资源管理器中定位文件
- **复制内容** / **另存为**

### 成果物面板

点击标题栏的 📎 按钮打开成果物面板，可以查看 AI 生成的所有文件：

- **全部**：所有生成的文件
- **文档**：Word、PDF 等文档类文件
- **代码**：脚本、源代码文件
- **图片**：生成的图片
- **数据**：Excel、CSV 等数据文件
- **媒体**：音频、视频文件

## 12. 设置面板

点击侧边栏底部的 ⚙️ 按钮打开设置面板，包含 5 个标签页：

### 客户端设置

- **输出目录**：AI 生成文件的默认保存位置
- **Debug 模式**：开启后在底部显示实时日志（故障排查用）
- **语音设置**：自动朗读、语音角色选择
- **界面语言**：中文 / English 切换

### 服务端设置

- **模型配置**：选择编排模型、执行模型、嵌入模型
- **供应商密钥**：管理各厂商的 API Key
- **Web 搜索**：配置搜索引擎 API Key
- **MCP 外部工具**：添加/管理扩展工具（如 Excel、PPT 工具）
- **沙盒隔离**：代码执行的安全模式选择

### 记忆管理

详见 [长期记忆](#8-长期记忆) 章节。

### 智能体设置

- **助手名称和角色设定**：全局生效的人设配置
- **独立模型配置**：为每个 Agent 分别指定不同的模型
- **技能管理**：添加、编辑、启用/禁用技能

### 云端设置

- **OpenFlux 云端账户**：登录/登出 NexusAI 云端
- **Router 消息路由**：配置 Router 连接参数
- **托管配置**：使用 Router 统一管理的模型和 API Key

## 13. 云端与远程操控

### 独立使用（默认）

OpenFlux 默认独立运行，无需任何云端服务。你只需要配置 LLM API Key 就能使用全部功能。

### 连接 Router

连接 OpenFlux Router 后，你可以通过**飞书、微信**等平台远程与 AI 对话：

1. 在 **设置 → 云端 → Router 消息路由** 中填入 Router 地址
2. 启用连接
3. 绑定配对码

连接成功后，在飞书群或私聊中发消息就会转发给你桌面端的 OpenFlux 处理。

### 托管模式

如果你的团队管理员在 Router 上配置了共享模型，你可以直接使用，无需自己配置 API Key：

在 **设置 → 云端 → 托管配置** 中开启"使用托管配置"即可。

## 14. 使用技巧与最佳实践

### 💡 对话技巧

1. **描述越具体，结果越好**
   - ❌ "帮我写个脚本"
   - ✅ "用 Python 写一个脚本，读取 D:\data\sales.csv，统计每月销售额并画折线图"

2. **拆解复杂任务**
   - 一次性说太多，不如分步骤来
   - "先帮我整理需求 → 然后写代码 → 最后测试"

3. **善用 @ 指定助手**
   - `@coder` 处理代码任务更专业
   - `@automation` 处理浏览器和定时任务更专业

4. **利用文件拖放**
   - 直接把文件拖进对话框让 AI 分析
   - 支持图片、文档、代码文件等

### 💡 定时任务技巧

1. **用自然语言就好**
   - 不需要记任何格式或命令
   - "每周五下午 5 点" 比 "cron 0 17 * * 5" 更好用

2. **适合做定时任务的场景**
   - ⏰ 定时提醒：开会、打电话、吃药
   - 📊 定期报告：每日日志检查、每周数据汇总
   - 🔍 定期监控：检查服务器状态、监控文件变化
   - 📧 定期发送：每天发简报、每周发周报

### 💡 技能学习技巧

1. **先让 AI 学技能，再做任务**
   - 学会的技能比临时回答质量高得多
   - 适合需要**反复使用**的专业方法论

2. **推荐学习的技能方向**
   - 深度研究：适合资料搜集和分析
   - 代码审查：适合团队代码质量管控
   - PPT 制作：适合经常做汇报的人
   - 数据分析：适合经常处理表格的人
   - 文案写作：适合内容创作者

3. **自定义技能更灵活**
   - 在设置里手动添加技能，可以写入你自己的专业知识和工作流程
   - 比如你公司的代码规范、文档模板、审核流程等

### 💡 工作流技巧

1. **关键词匹配**
   - 说"初始化项目"会自动进入项目创建流程
   - 说"代码审查"会自动进入审查流程
   - 不需要记工作流名称，说相关的话就行

2. **自动检测**
   - AI 会根据你的描述自动判断要不要使用工作流
   - "在 D:\new 下创建一个 Node 项目" → 自动触发项目初始化

### 💡 记忆技巧

1. **主动告诉 AI 你的偏好**
   - "我喜欢简洁的回答风格" → AI 会记住
   - "我的项目用 Python 3.11" → AI 之后会默认使用 Python 3.11

2. **搜索记忆验证**
   - 在记忆管理中搜索关键词，可以确认 AI 是否正确记住了信息

### 💡 浏览器自动化技巧

1. **分步描述复杂操作**
   - "先打开京东 → 搜索机械键盘 → 按价格排序 → 把前 5 个商品信息整理成表格"

2. **适合自动化的任务**
   - 重复性的网页操作
   - 数据抓取和对比
   - 表单批量填写

## 15. 常见问题

### 发送消息后一直"思考中"怎么办？

- 检查网络连接是否正常
- 确认 API Key 是否有效（设置 → 服务端 → 供应商密钥）
- 如果使用的是免费额度，可能已用完
- 尝试切换其他模型供应商

### 语音功能不可用？

- 语音识别需要下载语音模型，首次使用需等待
- 确保麦克风权限已授权
- 语音合成（朗读）只需联网即可使用

### 浏览器操作看不到窗口？

- 检查设置中浏览器的 headless 模式是否关闭
- headless 关闭 = 可看到浏览器窗口

### AI 不记得之前说过的话？

- 同一个会话内 AI 有上下文记忆
- 跨会话的记忆依赖"长期记忆"功能，需在配置中启用
- 在记忆管理中确认信息是否已保存

### 定时任务没有执行？

- 确保应用保持运行（最小化到托盘也可以）
- 在定时任务面板检查任务状态是否为"活跃"
- 查看执行记录确认是否有错误

### 如何切换语言？

在 **设置 → 客户端 → 界面语言** 中选择中文或 English，立即生效。

### 数据存在哪里，安全吗？

- 所有数据都保存在**本地电脑**上
- 对话历史在项目的 `sessions/` 目录
- 长期记忆在 `openflux_memory.db` 文件中
- 不会上传到任何第三方服务器
- 只有 LLM API 调用时消息会发送给你选择的模型供应商
